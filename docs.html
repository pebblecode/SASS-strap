Introduction

Sassap is an alternative way of using Twitter Bootstrap built with SASS & Compass

Sassap aims to solve a few of the problems I see with Twitter Bootstrap namely...

  • non-semantic: Bootstrap require you to ass classes like .row-fluid to your markup. I would much rather use one or two semantic classes per element than lots of different generic styling / layout classes
  • redundant: Bootstrap can do a lot of different things. There is a lot of functionality that I will never use and lots more that I will only use rarely. I would like to decide what parts of bootstrap on a project by project basis.
  • pre-styled: Bootstrap provided a lot of pre-styled elements such as buttons. This is good but to customize these elements you either have to write CSS on top of bootstrap that overrides Bootstrap (which learns to lots of unnecessary CSS) or you have to edit Bootstrap. I would prefer to style elements directly.

Sassap semantics

Sassap uses @extend to apply styles to elements in SCSS rather than applying the classes.

Bootstrap original html. no semantic classes no css required.

    <article class="row-fluid">
      <p class="span4">Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>
      <p class="span4">Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>
      <p class="span4">Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>
    </article>

Sassap html with semantic class names

    <article class="sassap-intro">
      <p>Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>
      <p>Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>
      <p>Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>
    </article>

Sassap SCSS using @extend

    .introduction{
      @extend .row-fluid;

      p{
        @extend .span4;
      }
    }
